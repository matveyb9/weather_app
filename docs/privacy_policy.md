# Политика конфиденциальности

**Приложение «Погода»**  
Последнее обновление: март 2025 г.

---

## 1. Общие положения

Настоящая Политика конфиденциальности описывает, какие данные собирает и использует приложение «Погода» («Приложение»), разработанное и поддерживаемое независимым разработчиком.

Мы придерживаемся принципа минимальной необходимости данных: Приложение собирает только те сведения, которые требуются для его непосредственной работы.

---

## 2. Какие данные мы НЕ собираем

Приложение **не собирает, не хранит и не передаёт**:

- имена, email-адреса или любые другие идентификаторы пользователя;
- историю поиска городов;
- данные об использовании приложения, аналитику, телеметрию;
- рекламные идентификаторы (IDFA, GAID);
- финансовую информацию;
- данные с микрофона, камеры или контактов.

Приложение не содержит рекламы и не интегрировано ни с одним аналитическим SDK.

---

## 3. Данные, обрабатываемые локально на устройстве

Следующие данные хранятся **исключительно на устройстве** пользователя и не покидают его:

| Данные | Назначение | Где хранится |
|--------|------------|--------------|
| Название последнего выбранного города и его координаты | Восстановление состояния при следующем запуске | SharedPreferences устройства |
| Кэш последнего ответа API погоды | Отображение данных в офлайн-режиме | SharedPreferences устройства |
| Настройки единиц измерения и темы | Сохранение пользовательских предпочтений | SharedPreferences устройства |
| Данные для виджетов рабочего стола | Отображение погоды в виджете | App Group (только Android и iOS) |

Все локальные данные удаляются при удалении Приложения.

---

## 4. Передача данных третьим сторонам

Для работы Приложение обращается к следующим сторонним сервисам:

### Open-Meteo (api.open-meteo.com)
- **Что передаётся:** GPS-координаты или координаты выбранного города.
- **Зачем:** получение данных о погоде.
- **Политика Open-Meteo:** [https://open-meteo.com/en/terms](https://open-meteo.com/en/terms)

### Open-Meteo Geocoding (geocoding-api.open-meteo.com)
- **Что передаётся:** текстовый запрос (название города).
- **Зачем:** поиск координат по названию.

### Nominatim / OpenStreetMap (nominatim.openstreetmap.org)
- **Что передаётся:** GPS-координаты.
- **Зачем:** определение названия населённого пункта по координатам (обратное геокодирование).
- **Политика Nominatim:** [https://nominatim.org/release-docs/latest/api/Overview/](https://nominatim.org/release-docs/latest/api/Overview/)

Все три сервиса являются открытыми и не требуют авторизации. Разработчик Приложения не контролирует и не несёт ответственности за политику конфиденциальности этих сервисов.

---

## 5. Геолокация

Приложение запрашивает разрешение на доступ к местоположению устройства только при явном действии пользователя (нажатии кнопки «Определить местоположение»).

- Разрешение является **опциональным**. Приложение полностью функционирует без него.
- На **Linux** геолокация недоступна технически — кнопка скрыта.
- Координаты используются однократно для запроса погоды и не сохраняются в постоянное хранилище в виде «истории местоположений».

---

## 6. Фоновая активность (Android и iOS)

На Android и iOS Приложение может запрашивать данные погоды в фоновом режиме для обновления виджетов рабочего стола.

- Фоновый запрос выполняется с теми же параметрами (координаты последнего города), что и основной.
- Никакие дополнительные данные при этом не собираются.
- Фоновая активность работает только при наличии активного интернет-соединения.

---

## 7. Безопасность данных

Поскольку Приложение не передаёт данные на собственные серверы и не использует авторизацию, основные риски безопасности отсутствуют. Данные виджетов на iOS защищены механизмом App Group с ограничением доступа по Team ID.

---

## 8. Дети

Приложение не предназначено специально для детей и не собирает никаких данных, которые могли бы идентифицировать ребёнка.

---

## 9. Изменения политики

В случае существенного изменения настоящей Политики обновлённая версия будет опубликована в репозитории с обновлённой датой в заголовке. Мы рекомендуем периодически проверять актуальность документа.
